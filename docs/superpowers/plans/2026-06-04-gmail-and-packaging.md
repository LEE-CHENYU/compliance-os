# Gmail Gating + `.dxt` v2 Packaging Implementation Plan (Plan 4 of 4)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the local-first extension shippable and well-behaved: (1) when Guardian's own Gmail OAuth isn't configured, the Gmail tools cleanly defer to the user's own assistant Gmail connector + the local upload path instead of hanging on a browser OAuth flow; (2) the `.dxt` package installs **local mode** with a single license-key field, no OpenAI key, no hosted API URL.

**Architecture:** Gmail is a *detected ambient capability* ‚Äî `gmail_client.is_gmail_configured()` checks for local Guardian OAuth creds; a `_gmail_guard()` helper short-circuits each of the 6 Gmail tools with a redirect message when not configured (the local-first default), so the user's Claude uses its own Gmail connector and feeds documents to `upload_document`. The `.dxt` manifest (generated by `scripts/build_dxt.py`) is bumped to v2.0.0: its env becomes `GUARDIAN_LICENSE_KEY` + `GUARDIAN_MODE=local`, and `user_config` shrinks to one required `license_key` field.

**Tech Stack:** Python 3.11, FastMCP, `zipfile`/`json` (stdlib), pytest. No new deps.

**Scope note (deferred to a follow-up Plan 5):** the hosted‚Üílocal **data migration** (`/api/dashboard/export` + `guardian-mcp import <zip>` with `user_id`/FK remapping + file rebasing) is intentionally NOT in this plan ‚Äî it is heavy, independent, and only serves *existing* cloud users (new users start local). The **`mcp_install.py` interactive installer** local-mode update (its multi-point prompt flow) is likewise deferred; the `.dxt` is the primary end-user rail.

def is_gmail_configured() -> bool:
    """True if Guardian's own Gmail OAuth is set up locally (a credentials

- [ ] **Step 4: Add `_gmail_guard()` and guard the 6 tools in `mcp_server.py`**

In `compliance_os/mcp_server.py`, add the guard helper right above `_get_gmail_service` (~line 1160):

```python
def _gmail_guard() -> str | None:
    """Return a redirect message (JSON) if Guardian's own Gmail isn't
    configured, else None. In the local-first default the user's assistant
    Gmail connector (or a Gmail MCP plugin) is the intended path ‚Äî Guardian
    ships no Gmail OAuth of its own."""
    from compliance_os.gmail_client import is_gmail_configured

    if is_gmail_configured():
        return None
    return json.dumps(
        {
            "error": "gmail_not_configured",
            "message": (
                "Guardian isn't managing Gmail here. Use your assistant's own "
                "Gmail connector (or a Gmail MCP plugin) to read or draft email, "
                "then save any attachment and call upload_document to bring it "
                "into your data room. To use Guardian's built-in Gmail "
                "integration instead, add OAuth credentials at "
                "~/.config/guardian/gmail_credentials.json."
            ),
        }
    )
```

Then add this **as the first statement in the body** (immediately after the docstring) of EACH of these 6 tools ‚Äî `gmail_search`, `gmail_read`, `gmail_draft`, `gmail_send`, `gmail_reply`, `gmail_download_attachment`:

```python
    if (msg := _gmail_guard()) is not None:
        return msg
```
